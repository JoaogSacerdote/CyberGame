# COM port 19 → 17 em settings.json

## Motivo
Usuário trocou de placa física. Placa nova reconhecida pelo Windows em COM17 (placa antiga era COM19). O `idf.portWin` (flash) já estava em COM17, mas `idf.monitorPort` ficou em COM19, fazendo a extensão ESP-IDF do VS Code tentar monitorar uma porta que não existe.

## Antes
```json
"idf.portWin": "COM17",
"idf.monitorPort": "COM19"
```

## Depois
```json
"idf.portWin": "COM17",
"idf.monitorPort": "COM17"
```

## Resultado
Monitor passou a abrir corretamente. Reload window do VS Code necessário pra extensão pegar a mudança.

**Nota:** `.vscode/settings.json` é untracked (não está no Git), então sobreviveu ao `git reset --hard` da entrada T1815.
